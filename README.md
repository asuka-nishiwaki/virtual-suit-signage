# バーチャルスーツサイネージ

**360°バーチャル試着体験　スムーズに試着し、納得できるスーツ選びを。**

React + Vite 製の Web プロトタイプ。店舗サイネージ想定の縦型 UI で、4方向撮影 → AI 診断 → バーチャル試着 → スマホ転送まで一気通貫で体験できます。

**公開 URL:** [https://orange-plant-0b4dcae00.7.azurestaticapps.net](https://orange-plant-0b4dcae00.7.azurestaticapps.net)

---

## 概要

### 解決したい悩み

- **他人視点での見え方**が分からない
- **客観的なサイズ感**が掴めない
- **実際の着用シーン（TPO）での印象**を事前に確認したい

### コンセプト

**「AIとデジタルで、より客観的にリアルにスーツを着た自分を」**

### ターゲット

- 絶対にスーツ選びに失敗したくない方
- スムーズに試着を済ませたい方

---

## 体験のコア / Do / Don't

### 体験のコア

**「試着がスムーズに、絶対にスーツ選びに失敗しない」**

→ **再定義後:** **「スムーズに試着し、納得できるスーツ選びを」**

撮影から試着・転送までを短時間で完結し、客観情報（サイズ・印象・TPO）をもとに納得して選べる体験を目指しています。

### 施策の Do


| #   | 施策                   | プロトタイプでの実装                                                                     |
| --- | -------------------- | ------------------------------------------------------------------------------ |
| ①   | **TPOに合わせた背景と照明の再現** | 屋外（ビル街）・オフィス・ホテルロビー・会議室 × 蛍光灯 / 暖色灯 / 太陽光フィルター                                 |
| ②   | **アプリ連携**            | **再定義後:** スマホ連携で試着をスムーズに。その場で決められない人も家族と相談でき、長時間利用による混雑を回避（QR モックで結果転送 UI を実装） |
| ③   | **360°全方位からのスタイル確認** | 正面・右・後ろ・左の **4方向** からドラッグ操作で切替                                                 |


### 施策の Don't


| #   | 施策                  | プロトタイプでの対応                                                       |
| --- | ------------------- | ---------------------------------------------------------------- |
| ①   | **選択肢を多くし過ぎない**     | 各スーツ **3色** に限定                                                  |
| ②   | **他人から試着シーンを見られない** | **再定義後:** スマホで試着姿・診断結果を確認できるようにし、サイネージ前での長時間滞留を回避（半個室運用 + スマホ転送） |
| ③   | **AIによる断定を避ける**     | 「似合い度 %」で参考値として提示（断定表現は使わない）                                     |

## 主要機能


| 機能                 | 内容                              |
| ------------------ | ------------------------------- |
| **AI 診断**          | 骨格・体格・パーソナルカラー・推奨サイズ・印象・シーン適正   |
| **360° バーチャル試着**   | 正面・右・後ろ・左の 4 方向からスーツ合成を確認       |
| **カラー・サイズ・TPO 変更** | 3色バリエーション、YA4〜BE6 サイズ選択、背景・照明切替 |
| **スマホ連携**          | 診断結果・試着情報を QR で転送（モック）          |


---

## 体験フロー

```
1. 撮影   … ガイドに従い 4 方向から撮影（MoveNet + ピンク骨格ガイド）
2. 診断   … 骨格・サイズ・印象を瞬時に分析
3. 試着   … TPO・照明を変えてシミュレーション（360° 確認）
4. 転送   … 結果をスマホへ（QR モック）
```

## 技術構成（リポジトリの解説）

### 技術スタック


| 区分      | 採用技術                                                                 |
| ------- | -------------------------------------------------------------------- |
| フロントエンド | **React 19** + **TypeScript** + **Vite 8**                           |
| スタイリング  | カスタム CSS（`pop-theme.css`、M PLUS Rounded 1c）                          |
| 姿勢推定    | **TensorFlow.js** + **MoveNet**（`@tensorflow-models/pose-detection`） |
| 試着合成    | Canvas 2D + PNG スプライトシート（`suitRenderer.ts`）                          |
| 診断      | フロントエンド mock + 任意で `/api/diagnose` バックエンド                            |
| ホスティング  | Azure Static Web Apps 想定（`npm run build` → `dist/`）                  |


> **Note:** GD 資料では Tailwind CSS / Lucide React 等も検討されていましたが、本提出プロトタイプは **Vite + 素の CSS** で POP デザインを実装しています。

### 起動方法

**重要:** `npm run dev` は `**frontend/` フォルダ内** で実行してください。  
ホームディレクトリ（`C:\Users\asuka` 等）で実行すると `Missing script: "dev"` になります。

```powershell
# プロジェクトルートへ移動
cd "C:\Users\asuka\OneDrive\ドキュメント\プログラミング\プログラミング\博報堂プロダクツ　デジタルサイネージ\博報堂プロダクツ　github\frontend"

# 依存関係インストール（初回のみ）
npm install

# 開発サーバー起動
npm run dev
# → http://localhost:5173 （ポート使用中の場合は 5174 等）

# 本番ビルド
npm run build
npm run preview
```

**PowerShell で `npm` が実行できない場合**（実行ポリシーエラー）:

```powershell
npm.cmd run dev
```

## 今後の課題

今回は **技術的なプロトタイプ** のため、スーツのフィッティング位置にズレが残る場合があります。本番環境では **高精度な骨格検知・店舗計測データとの連携** により解消する計画です。

その他の改善候補:

- 実 API による診断・在庫連携
- QR 経由の本番スマホ試着ページ
- 専用サイネージ端末・半個室レイアウトへの最適化
- HDR / 実写 360° 背景との統合
